I have a Goals page in my LogSeq vault that tracks the three big things I'm working toward. Not a task list -- more like the document I keep coming back to for direction. The weekly review references it, the morning check-in priorities should trace back to it, and the evening reflection is basically asking "did any of these move forward today?"

## Template

Three goals. I've tried having more and it doesn't work -- you just end up spreading thin and not making real progress on anything. Each one has a target, where I currently am, subgoals I can actually check off, and whatever context is useful.

```markdown
# Goals
The big three. Everything else supports these.

## 1. [Goal Name] — [Target]
**Current:** [where you are now]
**2026 Target:** [specific measurable target]
**Ultimate:** [longer-term version if applicable]
**Key Insight:** [the one thing that matters most for progress]

### Subgoals
- [ ] [Milestone 1] — [status]
- [ ] [Milestone 2] — [status]
- [ ] [Milestone 3] — [status]
- [ ] [Ultimate target]

### Supporting Context
[Training plans, key resources, relevant notes -- whatever helps]

---

## 2. [Goal Name] — [Target]
...same structure...

---

## 3. [Goal Name] — [Target]
...same structure...

---

## Weekly Review Checklist
- [ ] Check progress on each goal's subgoals
- [ ] Update any blockers or changes
- [ ] Set focus for coming week
```

## Setup Prompt

```
Create a Goals page in my vault. Use type:: system so it's
distinct from notes and reference material.

I want to track 3 major goals. For each one, include:
- The goal name and target
- Current state (where I am now)
- 2026 target (specific and measurable)
- Key insight (the one lever that matters most)
- Subgoals as checkboxes (milestones on the way to the target)
- Any supporting context (plans, resources, links)

At the bottom, add a weekly review checklist for checking
progress on each goal.

My three goals are:
1. [Goal 1]
2. [Goal 2]
3. [Goal 3]
```

## How It Connects

This page feeds into basically everything else in the Life OS:

- **[Weekly Review](/life-os/weekly-review)** -- has a "Goal progress" section. The subgoal checkboxes give you something concrete to report on instead of vague feelings about progress.
- **[Monthly Review](/life-os/monthly-review)** -- zooms out on each goal over the past month.
- **[Morning Check-in](/life-os/morning-checkin)** -- if none of your top 3 priorities for the day relate to any of your goals, that's worth noticing.
- **[Evening Check-in](/life-os/evening-checkin)** -- "what got done today" is basically a daily goal progress check.

The weekly review checklist at the bottom is there so you actually update the page. Goals you don't review are goals you forget about.

## A Few Things I've Learned

The subgoals need to be specific. "Get better at running" doesn't work. "Sub-1:30 half marathon" works. You need to be able to look at it and know whether you've done it or not.

The "key insight" line has been more useful than I expected. For any goal, there's usually one thing that matters way more than everything else. Writing it down keeps me from getting distracted by the other stuff.

And update the "current state" field when things change. It's easy to forget, but looking back at where you were three months ago compared to now is genuinely motivating.
