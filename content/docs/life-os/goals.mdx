I keep a Goals page in my LogSeq vault that tracks the big things I'm working toward. It's not a task list -- it's more like a north star document that the rest of the system references. The weekly review asks about goal progress. The morning check-in priorities should map back to these. The evening reflection checks whether the day moved the needle.

## Template

I keep it to three major goals. More than that and nothing gets real focus. Each goal has a target, current state, subgoals with checkboxes, and any supporting context.

```markdown
# Goals
The big three. Everything else supports these.

## 1. [Goal Name] — [Target]
**Current:** [where you are now]
**2026 Target:** [specific measurable target]
**Ultimate:** [longer-term version if applicable]
**Key Insight:** [the one thing that matters most for progress]

### Subgoals
- [ ] [Milestone 1] — [status]
- [ ] [Milestone 2] — [status]
- [ ] [Milestone 3] — [status]
- [ ] [Ultimate target]

### Supporting Context
[Training plans, key resources, relevant notes -- whatever helps]

---

## 2. [Goal Name] — [Target]
...same structure...

---

## 3. [Goal Name] — [Target]
...same structure...

---

## Weekly Review Checklist
- [ ] Check progress on each goal's subgoals
- [ ] Update any blockers or changes
- [ ] Set focus for coming week
```

## Setup Prompt

```
Create a Goals page in my vault. Use type:: system so it's
distinct from notes and reference material.

I want to track 3 major goals. For each one, include:
- The goal name and target
- Current state (where I am now)
- 2026 target (specific and measurable)
- Key insight (the one lever that matters most)
- Subgoals as checkboxes (milestones on the way to the target)
- Any supporting context (plans, resources, links)

At the bottom, add a weekly review checklist for checking
progress on each goal.

My three goals are:
1. [Goal 1]
2. [Goal 2]
3. [Goal 3]
```

## How It Connects to Everything Else

The goals page isn't something you look at in isolation. It feeds into the rest of the Life OS:

- **[Weekly Review](/life-os/weekly-review)** -- has a dedicated "Goal progress" section where you check in on each goal. The subgoal checkboxes give you something concrete to report on.
- **[Monthly Review](/life-os/monthly-review)** -- zooms out on goal progress over the past month and sets the next milestone for each.
- **[Morning Check-in](/life-os/morning-checkin)** -- your daily priorities should connect back to these goals. If none of your top 3 priorities for the day relate to any of your goals, that's worth noticing.
- **[Evening Check-in](/life-os/evening-checkin)** -- "what got done today" is a chance to see if you moved any goal forward.

The weekly review checklist at the bottom of the goals page is a reminder to actually update it. Goals that don't get reviewed don't get worked on.

## Tips

- Keep it to 3 goals. Seriously. If everything is a priority, nothing is.
- Make the subgoals concrete and checkable. "Get better at running" is not a subgoal. "Sub-1:30 half marathon" is.
- Update the "current state" regularly so you can see progress over time.
- The "key insight" line is underrated. For any goal, there's usually one thing that matters way more than everything else. Write it down so you don't lose focus.
