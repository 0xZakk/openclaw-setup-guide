---
title: Zettelkasten Processing Skill
description: A two-pass pipeline to turn any source into atomic vault notes
---

This is the pipeline that takes any source material -- book highlights, articles, papers, YouTube videos, podcasts -- and produces structured vault notes:

1. **Reference note** -- summary of the source with metadata
2. **3-8 literature notes** -- individual atomic insights, cross-linked
3. **Journal entry** -- logged in today's daily journal
4. **Cross-links** -- connected to existing vault notes

It uses two passes. Pass 1 reads the source and outlines the key insights worth capturing. Pass 2 writes each note as a standalone atomic idea. A single pass tends to produce surface-level notes -- the two-pass approach forces deeper thinking.

## Setup Prompt

```
Install or create a zettelkasten skill. The workflow should be:

1. Pass 1 (Outline): Read the source material. Create an outline of
   3-8 key insights worth capturing as literature notes. Each should
   be a full sentence expressing the core idea.

2. Pass 2 (Write): For each outlined insight, write a standalone
   literature note (2-3 paragraphs, in my own words, with cross-links
   to related concepts). Create the reference note linking to all
   literature notes.

3. Connect: Search existing vault notes for related concepts and
   add cross-links.

4. Journal: Update today's daily journal with what was created.

Use sub-agents for Pass 2 to write notes in parallel. Use the main
session (Opus) for orchestration and Sonnet for sub-agents to
manage costs.

Rules:
- Literature note titles = full sentences expressing the core idea
- Notes should be standalone -- readable without the source
- Write in your own words, not quotes (use quotes sparingly)
- Cross-link to existing vault notes where relevant
- Wave size: ~3 sources at a time to avoid burning API limits
```

## Cost Management

Processing books into notes uses real tokens. A few things I learned:

- Use Sonnet for the sub-agents that write individual notes. Opus for orchestration only.
- Batch in waves of 2-3 sources at a time.
- Space waves out to stay within rate limits. I burned through a lot of quota early on before figuring out the right pace.
