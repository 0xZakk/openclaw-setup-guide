title: LogSeq/Obsidian Vault
description: A personal knowledge base with atomic, interlinked notes
---

This is a zettelkasten-style knowledge vault where every note is an atomic idea connected to other ideas via bidirectional links. Two types of notes:

- **Reference notes** -- one per source (book, article, paper). Summarizes the source, links out to individual insights.
- **Literature notes** -- one per idea. A standalone concept written in your own words. Cross-linked to related notes and back to the source.

The vault becomes a web of interconnected knowledge. It gets more valuable the more you add to it because new notes connect to existing ones. I've built mine up to 350+ literature notes across dozens of books and papers.

This works with either **LogSeq** or **Obsidian** -- both are local-first markdown editors with bidirectional linking. I use LogSeq. The method matters more than the tool.

For syncing between machines, I use **Syncthing** (free, peer-to-peer) instead of iCloud or Dropbox. iCloud in particular causes file locking issues that will drive you crazy.

## Setup Prompt

```
I use [LogSeq/Obsidian] as my knowledge vault at [path].

Set up a zettelkasten system:

- Reference notes: One per source (book, article, paper). Contains:
  summary, key concepts (linked to literature notes), key quotes,
  metadata (authors, date, URL).

- Literature notes: One per idea/insight. Standalone atomic ideas
  written in my own words (2-3 paragraphs). Linked back to the source
  and cross-linked to related notes.

- Daily journals: At [journal path]. Log what was added each day.

When I send you a source to process, create the reference note,
extract 3-8 literature notes, cross-link everything, and update
the daily journal.

Format rules:
- Literature note titles should be full sentences expressing the
  core idea (e.g., "Human training data gives AI a faster start
  but a lower ceiling")
- All cross-linking should be bidirectional where relevant
- No subdirectories -- all pages in a flat folder
```

## Syncthing Setup

If you want your agent to have direct access to the vault (highly recommended):

1. Install Syncthing on both machines
2. Share the vault folder bidirectionally
3. Set conflict resolution to "newest wins"

This gives the agent real-time read/write access without cloud sync headaches.
