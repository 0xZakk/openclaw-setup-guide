---
title: Heartbeat System
description: Periodic proactive check-ins for background monitoring
---

OpenClaw sends periodic "heartbeat" polls to your agent. On each heartbeat, the agent reads `HEARTBEAT.md` and does whatever tasks are listed. This is how your agent does proactive work without you explicitly asking.

Good heartbeat tasks: checking email for urgent messages, checking your calendar for upcoming events, reviewing notifications, doing background organization work.

## Setup Prompt

```
Set up HEARTBEAT.md in the workspace. When you get a heartbeat poll:

1. Read HEARTBEAT.md for any tasks
2. If tasks are listed, do them
3. If empty or only comments, reply HEARTBEAT_OK

Track what you last checked in memory/heartbeat-state.json:
{
  "lastChecks": {
    "email": <timestamp>,
    "calendar": <timestamp>,
    "weather": null
  }
}

Rules:
- Rotate through 2-4 checks per day (don't check everything
  every time)
- Stay quiet at night (11 PM - 8 AM) unless urgent
- Only reach out if something needs attention:
  - Important email arrived
  - Calendar event coming up (<2 hours)
  - Something interesting found
- Be quiet otherwise (HEARTBEAT_OK)
```

## Heartbeat vs. Cron

| Use Heartbeat When | Use Cron When |
|---------------------|---------------|
| Multiple checks can batch together | Exact timing matters |
| You need conversational context | Task needs isolation |
| Timing can drift slightly | One-shot reminders |
| Want to reduce API calls by combining checks | Output goes directly to a channel |

General rule: batch similar periodic checks into HEARTBEAT.md. Use cron for precise schedules and standalone tasks.
